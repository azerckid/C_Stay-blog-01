# 배포 전 미완료 항목 체크리스트

이 문서는 `docs/01_IMPLEMENTATION_PLAN.md`를 기반으로 배포 전에 구현되지 않은 항목들을 정리한 것입니다.

## 전체 Phase 현황

- **완료된 Phase**: Phase 0-12, 12.5 (약 90% 완료)
- **미완료 Phase**: Phase 13 (모바일 앱 배포), Phase 14 (테스트 및 최적화)

---

## 필수 구현 항목 (배포 전 권장)

### 1. Phase 12.5: 메시지 기능 - 완성도 제고

#### 1.1 에러 핸들링 ✅ (최근 완료)
- [x] API 에러 응답 처리 (`fetcher.data?.error` 체크)
- [x] Toast 알림을 통한 에러 표시
- [x] Optimistic UI 실패 시 롤백 처리

#### 1.2 남은 작업
- [ ] **Skeleton Loading**: 데이터 페칭 시 스켈레톤 UI (대화 목록, 프로필 정보 등)
- [ ] **대화방 상세 설정**: 알림 끄기, 대화 삭제, 사용자 차단/신고 메뉴 UI
- [ ] **모바일 제스처**: 스와이프 뒤로가기 제스처 대응
- [ ] **알림 시스템 통합**: 앱 전체 영역에서 Pusher 공통 채널 구독, 새 메시지 알림 배지 업데이트

**우선순위**: 중간 (기본 기능은 동작하지만 UX 개선 필요)

---

### 2. Phase 4: 트윗 기능 - 페이지네이션

- [ ] **페이지네이션 (무한 스크롤)**: 홈 피드 트윗 목록 페이지네이션
  - 현재 상태: 문서상 미완료로 표시되어 있으나, 검색 기능에는 무한 스크롤이 구현됨
  - 확인 필요: 홈 피드에 무한 스크롤이 실제로 구현되어 있는지 확인 필요

**우선순위**: 중간 (트윗이 많아질 경우 성능 이슈 가능)

---

### 3. Phase 7: 답글 시스템 - 답글 개수 표시

- [ ] **답글 개수 표시**: 트윗 카드 하단에 답글 아이콘과 개수 표시
  - 현재 상태: 답글 기능은 구현되었으나, 피드의 트윗 카드에 답글 개수가 표시되지 않음
  - 참고: 답글 알림은 Phase 12에서 이미 구현됨

**우선순위**: 낮음 (기능적으로 문제 없으나 UX 개선)

---

### 4. Phase 8.5: 여행 태그 기반 필터링

- 현재 상태: 문서상 일부 미완료로 표시되어 있으나, 실제로는 대부분 구현된 것으로 보임
  - ✅ 태그별 트윗 목록 페이지
  - ✅ 태그 클라우드/목록 UI (우측 사이드바)
  - ✅ 인기 태그 표시
  - 확인 필요: 실제 구현 상태 재확인

**우선순위**: 낮음 (대부분 완료된 것으로 추정)

---

## 선택적 구현 항목 (배포 후에도 가능)

### 5. Phase 12: 국내 특화 지도 서비스 (네이버 지도 연동)

- [ ] **네이버 지도 연동**: 하이브리드 지도 시스템 (해외: Google, 국내: Naver)
- [ ] **사용자 설정 옵션**: '국내 여행 시 네이버 지도 우선 사용' 옵션

**우선순위**: 낮음 (한국 전용 기능, 선택 사항)

## Phase 13: 모바일 앱 배포 (Capacitor)

전체 Phase가 미완료 상태입니다. 웹 배포 시 필수는 아니지만, 모바일 앱 배포를 원한다면 구현 필요합니다.

**작업 항목**:
- [ ] Capacitor 설치 및 설정
- [ ] React Router v7 SPA 모드 설정
- [ ] Capacitor 설정 (아이콘, 스플래시, 권한)
- [ ] iOS 앱 설정
- [ ] Android 앱 설정
- [ ] PWA 지원

**우선순위**: 낮음 (웹 배포 시 필수 아님, 모바일 앱 배포 시 필요)

---

## Phase 14: 테스트 및 최적화

전체 Phase가 미완료 상태입니다. 프로덕션 배포 전에 권장되지만, 필수는 아닙니다.

**작업 항목**:
- [ ] 테스트 설정 (Vitest 등)
- [ ] 단위 테스트
- [ ] 통합 테스트
- [ ] 성능 최적화 (이미지 최적화, 쿼리 최적화, 코드 스플리팅)
- [ ] 접근성 개선 (키보드 네비게이션, 스크린 리더, ARIA)
- [ ] 문서화 (API, 컴포넌트, 배포 가이드)

**우선순위**: 중간-높음 (프로덕션 배포 시 권장, 특히 성능 최적화와 접근성)

4. Phase 13의 모바일 앱 배포
